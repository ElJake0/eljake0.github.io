Project Idea

Overview

The aim of this IT project is to develop a smartphone app for Android that conveniently packages a set of tools and trackers that are designed to increase productivity and positive mental wellbeing for users. The app will track usage of each productivity tool and give a visual representation of the user’s progress at intervals requested by the user. Users will also be able to set goal benchmarks to strive for, and the app will provide tips and resources to assist those that are struggling or want more information about the benefits behind each technique.


Motivation

Behind this app is my own motivation for fitting more things into each day and the desire to stay relaxed while being as productive as possible. As a person who has struggled with mental health and motivation issues throughout my life, it is a desire of mine to assist anyone struggling in similar situations by providing them with the tools and knowledge that I have used to improve my own situation. As it says in the Dao De Jing: “a journey of a thousand miles begins with a single step.” It would be a significant thing to me to help people take their first step.


Description

Separated into two sections, the app will provide tools and techniques designed to target productivity and wellbeing. The techniques and tools will be based on well known and proven practices which in some cases have been used for thousands of years.

Pomodoro Technique Timer:

The Pomodoro Technique Timer will be a simple timing/stopwatch style function which takes a user defined time (the goal time to be committed to the task at hand), and then divides that time into four blocks. At the end of each of the smaller blocks a bell will chime, however on the fourth chime the user will be prompted to take a short break for 3-5 minutes before resuming activity and resetting the timer.

To Do List:

The To Do List will be a basic dot point list format with check marks that denote a task as completed or incomplete.

Eisenhower Matrix:

The Eisenhower Matrix wil provider the user with a 2x2 grid where the user can add text to each grid cell to nominate tasks that fit the criteria of each cell:

Important and Urgent Important but Not Urgent
Not Important but Urgent Not Important and Not Urgent

Organising tasks in this way aid in prioritisation of tasks.

Meditation Timer:

The Meditation Timer will be a simple countdown timer where the user sets a goal time and selects from a range of ambient noise tracks that play softly in the background behind the timer. The ambient sound will be collected from a sound library, and embrace moods such as rainy, blizzard, bird sounds, meditation bells etc.

Breathing Exercises:

Breathing Exercises will be chosen from a list of 3 techniques: Square Breathing, Resonant Breathing, and Wim Hof Method breathing. The app will present simple text instructions on how to perform each method comfortably, and then a metronome will tick at the user’s chosen tempo. The metronome will instruct the user when to inhale and exhale.


Built on the Python framework Kivy, the application will center around a simple time module inherent in Python and an audio output module that will provide the user with atmosphere, such as ambient noise, and cues, such as bell chimes, to notify the user about time left in their chosen practice.

Data will be saved in hidden user files for access when the user wishes to view previous records. The data will be recorded in a CSV file format, so that the user can have the ability to take their data and view or import it into other software should they choose to.

The application will take a modern and minimalistic approach to colour and layout, with a mostly monochromatic design centering around a single highlight colour.

Settings and customisation in the application will initially be very limited. Initially the only controls provided will be a choice of where your record file will be stored, and a choice between light mode or dark mode in terms of colour scheme.


Tools and Technologies

To build this app in Python a number of built-in and external libraries, and also the mobile hardware for testing will be required.

Computers: To develop the application on.

Mobile devices (phones/tablets): To test the application during development.

Preferred Python IDE: To build and execute our Python code, I prefer PyCharm.

Kivy: A Python library that is used to develop cross-platform applications. Kivy is compatible with Android, iOS, Windows, Linux, and MacOS, and provides flexible UI elements to keep the app looking modern.

Time: A built-in Python library that interfaces with the device’s internal clock.

OS: A built-in Python library that will enable the app to interact with the device’s file system. This will be required to save and retrieve any files that track the user’s progress.

playsound: A Python library used to output audio from a file.

Pandas: A Python library used for data analysis, this would be used in combination with the saved user progress files to show detailed graphs.


Skill Requirement

To complete this project it would be essential to be comfortable developing applications with Python and having a good amount of experience analysing data with Pandas. Developers would also need to be familiar with developing mobile applications and using Kivy as a base for their mobile applications.

In addition it would be necessary to have a decent knowledge of accessible UI/UX design and understand the technical limitations and differences between the different types of mobile devices and operating systems.

It would be beneficial but not essential for developers on this project to be familiar with audio file formats and conversions between formats when an audio source is provided in an incompatible format.

Finding developers with experience using Kivy may be difficult, however this is balanced by the plethora of developers with experience in Pandas and the free nature of Python related development software. The availability and cheap cost of mobile devices will make this project easy to test.


Outcome

At the end of this project a functional mobile application will be produced that can provide users with some basic assistance in their daily life though enabling and encouraging healthy productivity and mental health habits. Users who take advantage of the methods provided in the application will be able to increase their output and feel more at ease while moving through daily life.

As the project itself is ultimately quite a simple framework it would serve as a great foundation from which to build in extra features and productivity methods, such as including licence free lectures from productivity experts.
